Logical Operators

combine boolean values so that you can express more complex ideas

&& and, evaluates to true if the values on both are true
|| or, evaluates to true if either one is true
! not, works on just one value

let userLoggedIn = true;
let userPurchasedItem = false;
// test if both values are true
userLoggedIn && userPurchasedItem; // => false
// test if either value is true
userLoggedIn || userPurchasedItem; // => true
// invert the value of a variable
!userLoggedIn; // => false
!userPurchasedItem; // => true

short-circuiting
code stops early if the first value of the || operator is truthy.
returns the first truthy value, or the last value if no values are truthy
code stops early if the first value of the && operator is falsy.
returns the first falsy value, or the last value if no values are falsy.

Objects

An object holds many values and each value has a unique key
// an example of an object
let gifts = {
tommy: "stuffed giraffe",
lisa: "coloring book",
roberto: "baseball cap",
beth: "suit of armor",
}; //
key (or property): "value",

Objects allow you to keep track of the value that belongs to each key. Then, you can retrieve a value using the name of the key.

gifts["tommy"]; // 'stuffed giraffe'

Objects start and end with curly brackets: the { and }. Keys must be a string data type, but they can be any string. A colon : sits between each key and the value it points to, and a comma , separates key: value pairs. values can be any data type.

let categories = {

categories['best picture']; // 'Shrek'

Return

use it in front of the final result that you want to share outside of the function
as opposed to console.log(), which only shows in console and remains undefined, return allows the result to be referenced within the program

function returnsTen() {
return 10;
}
let eleven = returnsTen() + 1; // becomes 10 + 1
console.log(eleven); // 11

Whenever JavaScript sees return in a function, it:

stops running the code in that function
takes the value to the right of return
swaps in the return value where the function was called

function add(num1, num2) {
if (num1 < 0 || num2 < 0) {
return "Only positive numbers please!";
}
return num1 + num2;
}

console.log(add(-1, 4));

If you wish to style the email links differently than the rest of your page links, a unique pseudo class is available to target links that start with "mailto:".

// CSS STYLES
// Normal Link
a {
font-family: arial;
text-decoration: none;
color: blue;
}

// Email Link
a[href^="mailto:"]{
color: red;
}


CSS allows you to supply the keyword !important in order to make a rule, which might otherwise be lower in the cascade, override others. Here's an example:

p {
color: red !important;
}



There are four pseudo-class selectors for anchor elements:

a:link: This is the normal state of a text link.
a:visited: This is the state of a text link that has been visited already by the current web browser.
a:hover: This creates the roll-over state for when the user's mouse rolls over (aka hovers over) a link.
a:active: This affects the state of the link while the user is actively clicking on it.
Due to the cascading aspect of CSS, the order these pseudo-classes are written within the CSS page is very important.
They must be written in this order:

link
visited
hover
active
