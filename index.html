combine boolean values so that you can express more complex ideas && and, evaluates to true if the values on both are true || or, evaluates to true if either one is true ! not, works on just one value let userLoggedIn = true; let userPurchasedItem = false; // test if both values are true userLoggedIn && userPurchasedItem; // => false // test if either value is true userLoggedIn || userPurchasedItem; // => true // invert the value of a variable !userLoggedIn; // => false !userPurchasedItem; // => true short-circuiting code stops early if the first value of the || operator is truthy. returns the first truthy value, or the last value if no values are truthy code stops early if the first value of the && operator is falsy. returns the first falsy value, or the last value if no values are falsy.

Data Types

Strings - text or numbers in " " or ' ' // declare a string with quotes let city = "Seattle"; let state = "Washington"; Numbers - Integers (whole numbers) and floating point decimal numbers (2.345) Booleans - True or False // declare a boolean let isLoggedIn = true; // evaluate a boolean if (isLoggedIn) { console.log("The user is logged in."); Pronounced 'not' or 'bang', ! inverts a boolean value. This means that !false turns into true, and !true turns into false. Null - is a data type that's used to indicate that a variable has no value. null is used as the default value when other data is not available. Even though null is a distinct data type, typeof will tell you that it's an object Undefined - is a special value that means that a variable was never defined. null value means that the developer decided that the value was empty on purpose. An undefined value means that the value was never assigned. you shouldn't assign a variable undefined. If you want to represent the absence of a value in your code, use null instead let example = null; example; // null let magicWord; magicWord; // undefined

Converting Data Types

If you combine a number with a string using the + operator, the resulting value will be a string. This process of automatic conversion is called coercion. But that's where parseFloat() comes in. toString() which turns a number into a string let age = 101; let ageString = age.toString(); typeof age; // 'number' typeof ageString; // 'string'

Objects

An object holds many values and each value has a unique key // an example of an object let gifts = { tommy: "stuffed giraffe", lisa: "coloring book", roberto: "baseball cap", beth: "suit of armor", }; // key (or property): "value", Objects allow you to keep track of the value that belongs to each key. Then, you can retrieve a value using the name of the key. gifts["tommy"]; // 'stuffed giraffe' Objects start and end with curly brackets: the { and }. Keys must be a string data type, but they can be any string. A colon : sits between each key and the value it points to, and a comma , separates key: value pairs. values can be any data type. // an Object with a mix of data types let actionFigure = { name: "Super Awesome Guy", heightInCentimeters: 10, accessories: ["cape", "disguise", "laser ring"], limitedEdition: true, }; you can get the values inside it using either dot notation or bracket notation actionFigure.name; // 'Super Awesome Guy' actionFigure.limitedEdition; // true let categories = { 'best picture': 'Shrek', 'best director': 'Shrek 2' }; categories['best picture']; // 'Shrek' categories['best director']; // 'Shrek 2' // adding a new key/value pair let myFamily = { mom: "Cynthia", dad: "Paul", }; myFamily.sister = "Lucinda"; // dot notation myFamily["brother in law"] = "Merle"; // bracket notation update existing value myFamily["brother in law"] = "Scotty"; objects within objects let car = { year: 2019, price: { suggested: 20000, retail: 15000, current: 12000, }, }; retrieve with double dot notation console.log(car.year.retail) //example of arry of objects with a loop checking them let partyPeople = [ { name: "Joe", attending: false }, { name: "Jasmine", attending: true }, { name: "Julio", attending: false }, { name: "Julia", attending: true }, ]; // loop through each item // check if the person is attending // if they are attending, say they are partying // otherwise, say they are burnt out for (let i = 0; i < partyPeople.length; i++) { if (partyPeople[i].attending) { console.log(partyPeople[i].name + " is partying!"); } else { console.log(partyPeople[i].name + " is burnt out on partying =("); } }

If you wish to style the email links differently than the rest of your page links, a unique pseudo class is available to target links that start with "mailto:".

// CSS STYLES
// Normal Link
a {
font-family: arial;
text-decoration: none;
color: blue;
}

// Email Link
a[href^="mailto:"]{
color: red;
}


CSS allows you to supply the keyword !important in order to make a rule, which might otherwise be lower in the cascade, override others. Here's an example:

p {
color: red !important;
}



There are four pseudo-class selectors for anchor elements:

a:link: This is the normal state of a text link.
a:visited: This is the state of a text link that has been visited already by the current web browser.
a:hover: This creates the roll-over state for when the user's mouse rolls over (aka hovers over) a link.
a:active: This affects the state of the link while the user is actively clicking on it.
Due to the cascading aspect of CSS, the order these pseudo-classes are written within the CSS page is very important.
They must be written in this order:

link
visited
hover
active
